# 좌석 점유 및 반납 정책

## 용어 정의
- **점유 (Hold)**: 기존 '선점' 대신 사용. 사용자가 좌석을 임시로 확보하는 상태. 코드 및 API에서는 `hold`를 사용합니다.
- **반납 (Release)**: 기존 '해제' 대신 사용. 점유한 좌석을 다시 비움. 코드 및 API에서는 `release` 또는 `return`을 사용합니다.
- **확정 (Occupied)**: 예약 및 결제가 완료되어 최종적으로 차지된 상태. `occupied`는 이 단계에서 사용합니다.

## 좌석 식별 방식
- **구간 (Block)**: "내야-연고-1"과 같은 형식의 이름으로 식별.
- **좌석 (Seat)**: 해당 구간 내에서의 행(row_number)과 열(col_number)로 식별.
- **표현 예시**: 구간 "내야-연고-1"의 1행 1열 좌석
