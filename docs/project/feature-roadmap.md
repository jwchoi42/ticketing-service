# 기능 로드맵 (Feature Roadmap)

## 1. 현재 구현된 기능

### 1.1. User (사용자)
- 회원가입 (`POST /api/users/sign-up`)
- 로그인 (`POST /api/users/log-in`)

### 1.2. Match (경기)
- 예매 가능한 경기 목록 조회 (`GET /api/matches`)

### 1.3. Site (좌석)
- **Hierarchy**: Area → Section → Block → Seat 계층 구조 조회
- **Allocation**: 좌석 선점(Hold), 반환(Release), 완료(Complete)
- **Status**: SSE 기반 실시간 좌석 상태 스트리밍

### 1.4. Reservation (예약)
- 예약 생성 (`POST /api/reservations`)

### 1.5. Payment (결제)
- 결제 요청/승인 (Toss Payment Gateway 연동)

### 1.6. Infrastructure
- SSE 실시간 통신
- Prometheus + Grafana 모니터링
- GitHub Actions CI/CD

## 3. 구현 로드맵

### Phase 1: 필수 기능 (MVP 완성)

사용자가 예매 후 관리할 수 있는 최소한의 기능 제공.

- [ ] 마이페이지 (예매 내역 조회)
- [ ] 예매 취소/환불
- [ ] 좌석 선점 타이머 UI

**목표**: 예매 → 확인 → 취소의 기본 사용자 여정 완성

### Phase 2: 경쟁력 확보

실제 서비스 수준의 완성도 확보.

- [ ] 모바일 티켓 (QR코드)
- [ ] 예매 확인 알림 (이메일/SMS)
- [ ] 좌석 등급 및 가격 차등
- [ ] 좌석 상세 정보 표시

**목표**: 실제 경기장 입장 가능한 티켓 발권

### Phase 3: 확장 기능

대규모 트래픽 대응 및 부가 수익 모델.

- [ ] 대기열 시스템
- [ ] 포인트/쿠폰 시스템
- [ ] 관리자 대시보드
- [ ] 시즌권/멤버십

**목표**: 대규모 트래픽 대응 및 운영 효율화

## 4. 기술적 고려사항

### 4.1. Phase 1 구현 시

| 기능 | 기술 고려사항 |
|------|---------------|
| 마이페이지 | 예약 목록 API, 프론트엔드 페이지 추가 |
| 예매 취소 | 환불 정책 로직, 결제 취소 API (Toss), 좌석 상태 복원 |
| 타이머 UI | `holdExpiresAt` 값 프론트엔드 전달 및 카운트다운 렌더링 |

### 4.2. Phase 2 구현 시

| 기능 | 기술 고려사항 |
|------|---------------|
| QR 티켓 | QR 코드 생성 라이브러리, 티켓 검증 API |
| 알림 | 이메일(SMTP/SendGrid), SMS(Twilio/NHN Cloud) 연동 |
| 가격 차등 | Seat 엔티티에 grade/price 필드 추가, 가격 계산 로직 |

### 4.3. Phase 3 구현 시

| 기능 | 기술 고려사항 |
|------|---------------|
| 대기열 | Redis Sorted Set 또는 Kafka 기반 순번 관리 |
| 포인트/쿠폰 | 별도 도메인 모듈, 결제 시 할인 적용 로직 |
| 관리자 | 별도 Admin 모듈 또는 권한 기반 분리 |
